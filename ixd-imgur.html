Imgur Upload Experience

As an image-sharing service, Imgur wants to make uploading and sharing images simpler for users, so that they can increase overall share of, and integration in, the social image-sharing business.

Problem Definition

The current image upload process lacks clear direction which confuses users. A successful upload process will take a new user and show them how to effectively upload and share images.

Audience

The project should focus on desktop users who upload images to share with others.

Team / Role

I was working alone on this project for both research and interaction design.

Constraints

I budgeted three days for this project and wanted to do both research and interaction design. My project plan was created in about three hours with research taking another five hours. The findings, analysis, and interaction design took about fifteen hours total.

Design Process

This project was broken into three phases: a task flow audit, goals analysis, and interaction redesign.

Understanding goals is at the core of any interaction, so before interface design begain I did a quick exploration of the goals an Imgur user might have.

I performed an informal poll of friends and colleagues who have shared images online in the last month and sorted the results into a chart comparing frequency of feature use to the number of users who reported usage. This helped me understand how to prioritize features.

After focusing on features I chose what to focus on when improving the interface, settling on two principles:

  • Instructional — the screen should show new users the easiest way to upload an image and help them understand what to do next.
  • Goal Oriented — The interface should prioritize a users primary focus, sharing images, and cluster related controls.
For each step in the upload process I performed an analysis and refined the interface to clarify what the user should do at each step.

Step 1 — Choosing the Files

When users arrive on the home screen and tap the upload button they are presented with a screen with four equally weighted options (Fig. 1.1).

Figure 1.1 - Original State

Figure 1.2 - Updated State

The trouble weighting options equally it’s not obvious which route is best. Imgur can take advantage of this opportunity to teach new users the best way to upload images.

I propose a new look (Fig. 1.2) which keeps the process and options intact, but focuses new users on Drag and Drop. The new interface is much easier to scan and understand which options are available.

Step 2 — Beginning the Upload

Once the user has selected files to upload, the Imgur interface still does not show clear hierarchy (Fig. 2.1). Information is competing for attention and is not clearly separated by task or order of operations. Some content is green which draws the eye, visual treatment of button areas and content areas are similar, and even reading through the non-linear content is difficult.

By de-emphasizing already-used controls (Fig. 2.2), hiding a few ambiguous controls (edit image & filename), and emphasizing next steps the interface what users should do next.

Figure 2.1 - Original State

Figure 2.2 - Updated State

Figure 2.3 - Original State w/ Gallery

I've ordered the page to provide an overview of the entire upload process:

  • Header Info - What user is doing and an escape hatch to abandon the upload
  • File Attachment Methods - Instructions to attach another file
  • Informational Message - Message about file upload state
  • Picture Info and Options - Information about specific file(s)
  • Upload Options / Start Upload - Select options and begin upload
Options like Make Album are still available (Fig. 2.3), and remain in chronological order. “Choose images, see chosen images, do something with images, and upload images.”

The focus is again on accomplishing user goals. Everything points the user toward Start Upload and everything is chunky and tappable for devices accessing via mobile web.

Step 3 — The Upload Begins

The current Imgur interface shows each file individually and repeats the “green bar fills to the right” process for each file. This does not show the user how many total files have uploaded or how many remain to upload.

Figure 3.1 - Original State

Figure 3.2 - Updated State

In the proposed screen I’ve kept all the previous functionality, and ensured the user knows which files have uploaded, which failed, and about how much longer it will be until the rest of the files have uploaded. There is also a message area that can display different system messages, followed by a clearly-labelled escape hatch.

Step 4 — The Upload Succeeds!

Now the upload process is coming to a close. In the Imgur interface, statistics, edit, description, views, posting time, etc are sprinkled about the page. Icons for social sharing are both below the picture and on the right side of the page. Share in Imgur is also not an obvious choice.

Figure 4.1 - Original State

Figure 4.2 - Updated State

In the proposed page, the original functionality exists but has been reorganized on the page. I’ve put the title / description above the image, clustered with editing and deletion controls so operations which affect the image are all in the same area of the page. Information about photo views, posting time, and stats are in the upper right to balance the container, again to aid findability.

Sharing is only available in the right panel with Imgur’s sharing prioritized, simple social sharing highlighted, and image-size controls surfaced. The existing links are also displayed below the simple share options to ensure users who need specific links are still served.

Retrospective

I feel this project was a success, and resulted in a much better upload interaction for novice Imgur users. Upon sharing the interface with two of my original task analysis participants they understood what to do immediately.

If I could budget more time for this project I would like to do a full comparative usability test phase to determine the effectiveness of the updated interface.
